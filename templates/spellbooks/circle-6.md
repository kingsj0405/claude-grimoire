🧠 멘탈 모델

세션이 끝날 때마다 맥락이 증발한다면 속도가 나지 않는다.
**파이프라인**을 만들어라:
한 세션의 출력이 다음 세션의 입력이 되는 구조.
사람은 시작만 누르고 기다린다.

─────────────────────────────────────

📜 CLAUDE.md에 추가할 규약 (복사 가능)

```markdown
## 핸드오프 프로토콜
- Session A: 출력을 docs/output-a.json에 쓴다
- Session B: docs/output-a.json을 읽어 처리, docs/output-b.json에 쓴다
- Session C: 검증, docs/final.json에 승인 기록
- 각 단계는 독립 세션 + 독립 시각에 실행 가능

## 체크포인트
- 각 세션은 진입 시 이전 단계 결과물을 먼저 읽는다
- 실패 시 .failed 마커 파일을 남긴다
```

─────────────────────────────────────

⚡ 승급의 열쇠

"사람이 세션 A에서 B로 직접 복붙하고 있다면,
 그건 파이프라인이 아니다."

─────────────────────────────────────

🔮 승급 의식

1. 3단계 파이프라인(수집 → 처리 → 검증)을 구축한다.
2. 전 단계 병렬 실행으로 처리량을 측정한다.
3. 이 의식을 3회 반복한다.
