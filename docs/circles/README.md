# 서클 레퍼런스

각 서클의 **마법책 전문**은 플러그인 템플릿에 있습니다:

`../../templates/spellbooks/circle-{1..10}.md`

또는 플러그인 설치 후 Claude Code에서:

```
/grimoire book <N>
```

## 서클 요약 (1~10)

| 서클 | 이름 | tmux 세션 | 핵심 전환점 |
|------|------|-----------|-------------|
| 1 | 견습 마법사 | 단일 | Claude Code를 도구로 인식 |
| 2 | 서기관 | 단일 + CLAUDE.md | 반복 지시를 문서화 |
| 3 | 이중 시전자 | 2 병렬 | 작업 간 의존성/독립성 판별 |
| 4 | 바람의 직조사 | 3-4 병렬 | 작업을 DAG로 시각화 |
| 5 | 성좌 술사 | 5 + 감시 | 감시자/리뷰어 세션 전용 배치 |
| 6 | 시간 굴절사 | 6-7 + 파이프라인 | 세션 간 핸드오프 자동화 |
| 7 | 차원 설계사 | 7-8 + 메타 | Claude가 Claude를 지시 |
| 8 | 혼돈의 지배자 | 8-9 + 복구 | 실패를 전제로 한 설계 |
| 9 | 세계수의 현자 | 9-10 + 다중 프로젝트 | 개인 도구 → 팀 인프라 |
| 10 | 대마법사 | 10+ 풀 오케스트레이션 | 새 워크플로우 발명 |

## 채점 루브릭 (요약)

5개 차원, 총점 0~100, 10점 단위로 서클 매핑:

| 차원 | 최대 |
|------|------|
| CLAUDE.md 성숙도 | 25 |
| 병렬 세션 | 30 |
| 자동화 | 20 |
| 작업 구조화 | 15 |
| 협업 | 10 |

자세한 기준은 `../GUIDELINES.md` 또는 `/grimoire scan` 출력을 참고.
