🧠 멘탈 모델

사고가 "순차 실행"에서 **방향 비순환 그래프(DAG)**로 전환된다.
3-4개의 세션을 띄우되, 각 세션의 경계가 명확해야 한다.
"이 세션은 무엇을 하고, 저 세션은 무엇을 하는가"가
한 문장으로 정의되지 않으면 아직 준비가 안 된 것이다.

핵심 질문: "이 작업들 중 어떤 것이 서로에게 의존하는가?"

─────────────────────────────────────

📜 CLAUDE.md에 추가할 규약 (복사 가능)

```markdown
## 세션 역할 정의
- 세션 A (tmux 0): 핵심 구현 (src/)
- 세션 B (tmux 1): 테스트 작성 (tests/)
- 세션 C (tmux 2): 문서화 (docs/)
- 세션 D (tmux 3): 리팩토링/린팅

## 세션 간 규약
- 동일 파일을 두 세션이 동시에 수정하지 않는다
- 세션 A가 새 함수를 만들면, 세션 B가 해당 함수의 테스트를 작성한다
- 브랜치 전략: 세션별 feature 브랜치 분리
```

🔮 승급 의식

1. 하나의 기능을 4개 하위 작업으로 분해한다.
2. 각 tmux 창에서 독립적으로 Claude Code를 실행한다.
3. 완료 후 git merge로 통합하고 단일 세션 대비 시간 절약을 측정한다.
4. 이 의식을 3회 반복한다.
